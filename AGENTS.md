# AGENTS.md

This is the canonical instruction file for the repo. Other agent-specific instruction files should point here.

- The actual app lives in `HuMotivatoren/`; run app commands from there.
- Use `npm install` once in `HuMotivatoren/` to install all workspace deps.
- OpenCode should load `.github/agents/squad.agent.md` and use `.squad/team.md` plus `.squad/routing.md` as the squad source of truth.
- `npm run dev` starts backend and frontend together.
- `npm run build` runs `frontend` build first, then `backend` build.
- `npm run test` only runs the `tests` workspace. Use `npm run test --workspace=frontend` or `npm run test --workspace=backend` for package-specific tests.
- There is no repo-level lint script or formatter config in the checked-in manifests.
- Frontend dev uses Vite on `http://localhost:5173` and proxies `/api` to `http://localhost:3001`.
- Frontend API calls default to `VITE_API_URL || http://localhost:3001`.
- Backend loads `.env` in `backend/src/index.ts`, and `backend/src/app.ts` reads `CORS_ORIGIN` as a comma-separated allowlist.
- Backend env fallback is local-only: `PORT` defaults to `3001`, and `getMotivation` returns fallback content when Azure OpenAI vars are missing or the call fails.
- `backend/data/development-history/entries/*.md` and `backend/data/development-history/index.json` are generated by `.github/workflows/generate-change-entry.yml` on pushes to `main`.
- The development-history UI renders markdown with a small custom parser, so only headings, bold, inline code, fenced blocks, lists, and horizontal rules are supported.
- `squad-heartbeat.yml` says squad workflow updates are duplicated and should be synced with `squad upgrade`.
