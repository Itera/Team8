# refactor: extract LLM provider into service layer

**Hash:** 1e2f3a4  
**Date:** 2026-04-13  
**Author:** marcin  

## Summary

The LLM call logic that lived directly in the route handler was moved into a
dedicated `llmService.ts`. This makes the route handler thinner, improves
testability, and creates a clear boundary for swapping LLM providers.

## Changes

- Created `backend/src/services/llmService.ts` exposing `generateMotivation()`
- Updated `backend/src/routes/motivate.ts` to delegate to the service
- Updated unit tests to mock at the service boundary instead of the fetch level
- Removed duplicate error handling code that now lives in the service layer

## Notes

No behaviour change from the user's perspective. The refactor was driven by
the growing complexity of the route handler as new personality types were added.
